Limitation:
Structural fallback splitting can fragment retrievable context

The hybrid chunking pipeline adds a safety valve that subdivides any 
macro-chunk (from Claude's topic-boundary pass) that exceeds the size budget.
When the macro-chunk has no paragraph breaks to split on, common in PDF text where pdfplumber 
strips formatting — the valve falls back to splitting on sentence boundaries. This guarantees 
no chunk exceeds the size limit, but the sentence split is purely structural, with no awareness of meaning.

This tradeoff showed up directly in testing on the Bauerlein question 
("What specific claim does Bauerlein make about digital technology?") 
against the same 23,947-character corpus. Under the original chunking 
(4 chunks, one 13,630-character mega-chunk left intact), the pipeline correctly 
answered: "Bauerlein suggested that digital technology 'stupefies young Americans' (2009)," 
using 5,616 tokens. Under the hybrid chunking with the sentence-level safety valve (8 chunks, 
the same source text now split across 4 sub-chunks), the same question returned "I don't know. 
The context provided does not contain any claims made by Bauerlein about digital technology," using 2,498 tokens.

The fix reduced token cost by roughly 55% for this query, but cost retrieval accuracy on this specific question. 
The likely cause is that the sentence-level split separated the Bauerlein reference from the surrounding context that 
made it semantically retrievable as being "about digital technology" — once split, the relevant sub-chunk either ranked 
outside the top-k results or no longer carried enough contextual signal for the embedding model to surface it.

This is a direct illustration of the core RAG chunking tradeoff: smaller, size-compliant chunks are cheaper and reduce 
noise, but aggressive structural splitting (as opposed to semantic splitting) risks breaking apart facts that depend on 
their surrounding sentences for context. A possible next step is prepending a short context anchor (e.g. the original macro-chunk's heading)
 to each sentence-split sub-chunk, so fragments retain a link back to their source topic even after being split.
